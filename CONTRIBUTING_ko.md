# 기여 가이드

이 책에 관심을 가져주셔서 감사합니다! 촉각 센싱 및 로봇 공학 커뮤니티의 기여를 환영합니다.

## 기여 방법

이 프로젝트는 **Issue 기반 기여 워크플로우**를 사용합니다. PR을 직접 제출하는 대신, GitHub Issue를 통해 제안, 자료, 수정 사항을 제출합니다. 저자가 각 제출물을 검토한 후 승인된 콘텐츠를 책에 반영합니다.

### 1. 콘텐츠 제안 (주요 기여 경로)

본인의 자료, 센서 데이터, 제품 정보, 연구 노트, 전문 지식을 공유하고 싶으신가요?

- [**Suggest Content**](https://github.com/terryum/book-robot-hand-tactile-sensor/issues/new?template=suggest-content.yml) 템플릿을 사용하세요
- 본인의 자료를 제공해 주세요: Notion 페이지, 블로그 포스트, 제품 사진, 비교 데이터, 실사용 경험 등
- AI가 자료를 분석하여 초안을 작성하고, 저자가 검토 후 반영합니다
- 이것이 **Contributor**가 되는 주요 경로입니다 (아래 크레딧 정책 참조)

**가치 있는 기여의 예시:**
- 상용 센서 제품의 스펙, 사진, 실사용 경험 ([@sjchoi86](https://github.com/sjchoi86)의 촉각 센서 서베이 자료처럼)
- 미공개 연구 데이터나 실험 결과
- 학술 논문에서 찾을 수 없는 산업 현장의 인사이트
- 비교 표, 벤치마크, 실용 가이드

### 2. 오류 제보

오타, 사실 오류, 깨진 링크를 발견하셨나요?

- [**Report Error**](https://github.com/terryum/book-robot-hand-tactile-sensor/issues/new?template=error-report.yml) 템플릿을 사용하세요
- 챕터, 섹션, 해당 텍스트를 포함해 주세요

### 3. 번역 개선

한국어 또는 영어 번역이 어색하거나 부정확한 부분을 발견하셨나요?

- [**Translation Improvement**](https://github.com/terryum/book-robot-hand-tactile-sensor/issues/new?template=translation-fix.yml) 템플릿을 사용하세요

## 기여 워크플로우

```
기여자: Issue 작성 (템플릿 사용, 자료 첨부)
  → 저자: Issue 검토
  → 저자: AI 도움으로 콘텐츠 반영
  → 저자: 업데이트 배포 (책 + 웹사이트 + PDF)
  → README Contributors 테이블에 이름 추가
  → 커밋 링크와 함께 Issue 종료
```

## 크레딧 정책

### 누가 Contributor가 되나요?

README의 **Contributors** 테이블은 책을 풍부하게 만드는 **상당한 양의 콘텐츠**를 제공한 분들을 인정합니다.

**Contributor로 등재되는 경우:**
- 새로운 섹션이나 상당한 콘텐츠 추가로 이어지는 원본 자료(노트, 데이터, 사진, 제품 스펙)를 제공한 경우
- 출판된 문헌에서 찾을 수 없는 고유한 가치를 더하는 전문 지식이나 실무 경험을 공유한 경우
- 프로젝트 인프라에 사용되는 도구, 프레임워크, 디자인 패턴을 기여한 경우

**커밋 메시지에서 감사하되 Contributor로 등재되지 않는 경우:**
- 오타나 사소한 오류 제보 (커밋 메시지에서 크레딧)
- 소규모 번역 수정 제안 (커밋 메시지에서 크레딧)
- 추가 맥락 없이 공개된 논문이나 링크만 알려주는 경우

### Contributor 크레딧 방식

1. **README Contributors 테이블** — GitHub 아바타, 유저네임, 기여 내용이 기여량 순으로 표시됩니다
2. **챕터 참고문헌** — 콘텐츠가 제공된 자료에 기반한 경우, 해당 챕터의 참고문헌에 기여자 이름이 기록됩니다 (예: `Choi, S. (Korea University). (2026). Commercial tactile sensor survey.`)
